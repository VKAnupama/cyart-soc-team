## Overview
This task demonstrates both theoretical and practical knowledge of SOC activities. It covers alert prioritization, incident classification, the incident response lifecycle, and frameworks. The project also includes practical application of alert management, triage, documentation, and evidence preservation using tools like Wazuh, TheHive, VirusTotal, Velociraptor, and CrowdSec.

## Key Learnings
### Theoretical knowledge

**1. Alert Priority Levels:**
Understood the concept of alert priority levels
Learned the workflow of NIST SP 800-61
Analyzed real-world cases to map CVSS scores to priority levels

**2. Incident Classification:**
Learned about incident categories, taxonomy, and contextual metadata
Studied how to map incidents to MITRE ATT&CK tactics and techniques

**3. Basic Incident Response:**
Learned about the incident response lifecycle
Understood the procedures followed in handling incidents

### Practical Application

**4. Alert Management (Practical):**
Mapped alerts to MITRE ATT&CK techniques
Prioritized alerts based on severity
Created dashboards in Wazuh to visualize alert priorities
Drafted incident tickets in TheHive

**5. Response Documentation**
Used Google Docs to document incidents
Followed SANS templates
Documented investigation steps, checklists, and post-mortem reports

**6. Alert Triage:**
Analyzed alerts in Wazuh
Performed threat intelligence validation using VirusTotal

**7. Evidence Preservation:**
Installed and used Velociraptor
Collected volatile data
Performed memory dump collection

**8. Capstone Project:**
Understood how attacks are executed, detected, responded to, and documented
Learned how CrowdSec is used to block attackers' IP addresses

This project gives a basic understanding of how security incidents are detected, analyzed, and managed. It also highlights the use of different tools and frameworks in handling alerts and improving security operations.
